1. Genel Bakis ve Mimari

2. Veritabani Semasi (Prisma)
PostgreSQL uzerinde asagidaki tablolar kullanilacaktir:
2.1 Tablolar
// prisma/schema.prisma
model Account {
  id            String    @id @default(uuid())
  phone         String    @unique      // +905xxxxxxxxx
  label         String?               // Kullanici tanimi icin isim
  status        String    @default("disconnected")
  // connected | disconnected | banned | qr_pending
  isBackup      Boolean   @default(false)
  proxyHost     String?               // 9Proxy sticky session host
  proxyPort     Int?
  proxyUser     String?
  proxyPass     String?
  sessionData   String?   @db.Text    // Baileys session JSON
  dailySent     Int       @default(0)
  dailyLimit    Int       @default(200)
  lastSentAt    DateTime?
  createdAt     DateTime  @default(now())
  messages      Message[]
}
model Template {
  id        String    @id @default(uuid())
  name      String
  body      String    @db.Text   // {isim}, {soyisim} gibi degiskenler
  createdAt DateTime  @default(now())
  campaigns Campaign[]
}
model Campaign {
  id          String    @id @default(uuid())
  name        String
  templateId  String
  template    Template  @relation(fields: [templateId], references: [id])
  status      String    @default("pending")
  // pending | running | completed | failed
  totalCount  Int       @default(0)
  sentCount   Int       @default(0)
  failCount   Int       @default(0)
  createdAt   DateTime  @default(now())
  messages    Message[]
}
model Message {
  id          String    @id @default(uuid())
  campaignId  String
  campaign    Campaign  @relation(fields: [campaignId], references: [id])
  accountId   String
  account     Account   @relation(fields: [accountId], references: [id])
  toPhone     String
  body        String    @db.Text
  status      String    @default("queued")
  // queued | sent | failed
  error       String?
  sentAt      DateTime?
  createdAt   DateTime  @default(now())
}

Adim 5 — Baileys SessionManager
Tum WhatsApp hesaplarini ayakta tutan ana modul. src/whatsapp/SessionManager.js:
// SessionManager.js
// Gorev: Tum WA hesaplarini baslat, izle, yonet
class SessionManager {
  sessions = new Map()   // accountId → BaileysSocket
  // initialize()
  //   → DB'den tum aktif hesaplari cek
  //   → Her biri icin startSession() cagir
  // startSession(account)
  //   → ProxyManager'dan proxy al
  //   → Baileys socket olustur (SOCKS5 proxy ile)
  //   → QR gelince Socket.io ile panele gonder
  //   → Baglanti kurulunca DB'yi guncelle: status=connected
  //   → Ban/disconnect olunca: handleDisconnect() cagir
  // handleDisconnect(accountId, reason)
  //   → Neden ban mi, logout mu, network mi?
  //   → Ban ise: markAsBanned(), yedegi aktive et
  //   → Network ise: 30sn bekle, reconnect
  // activateBackup()
  //   → isBackup=true hesaplardan birini sec
  //   → isBackup=false yap, session'i baslat
  //   → Socket.io ile paneli bilgilendir
  // sendMessage(accountId, toPhone, message)
  //   → Ilgili session'i bul
  //   → baileys.sendMessage() cagir
  //   → Sonucu dondur
}
Adim 6 — BullMQ Mesaj Kuyrugu
Mesajlari kuyruga alan ve sirali gonderen sistem. src/queue/MessageQueue.js:
// MessageQueue.js
// Gorev: Kampanya mesajlarini kuyruga al, sirali gonder
// Kuyruk Ayarlari:
// - concurrency: 5  (ayni anda max 5 mesaj isle)
// - delay: her job icin 3000-8000ms random delay
// - attempts: 3     (basarisizlikta 3 kez tekrar dene)
// - backoff: exponential
// addCampaignToQueue(campaignId, messages[])
//   → Her mesaj icin ayri BullMQ job olustur
//   → Job'a: toPhone, body, accountId, campaignId
//   → Random delay ata (insan gibi gorunsun)
// processMessage(job)
//   → SessionManager.sendMessage() cagir
//   → Basarili: DB'de Message.status = sent
//   → Basarisiz: DB'de Message.status = failed, error kaydet
//   → Socket.io ile canli ilerleme gonder
// Account rotation:
//   → Round-robin: siradaki aktif hesabi sec
//   → Hesap gunluk limite ulastiysa atla
//   → Tum hesaplar limitli ise kuyrugu durdur, uyar

6. Onemli Teknik Kurallar
WhatsApp Ban Onleme
Her hesaba FARKLI sticky session proxy atanmali
Mesajlar arasi delay: 3-8 saniye RANDOM (sabit delay ban sebebidir)
Hesap basina gunluk max 200 mesaj (baslangicta 50'den baslat, kademeli artir)
Gece 00:00-08:00 arasi mesaj gonderme
Ayni numaraya ayni kampanyada 1'den fazla mesaj gonderme
Session dosyalari kaybolursa hesap yeniden QR taramak zorunda kalir — mutlaka DB'ye kaydet
Proxy Sticky Session
9Proxy'de GB-Based + Sticky Session kullan
Her Baileys socket'i proxy ile olustur:
import makeWASocket from '@whiskeysockets/baileys'
import { SocksProxyAgent } from 'socks-proxy-agent'
const agent = new SocksProxyAgent(
  `socks5://user:pass@proxy.9proxy.com:PORT`
)
const sock = makeWASocket({
  agent,   // tum trafik bu proxy uzerinden gider
  ...diger_ayarlar
})
Proxy dustu ise socket'i kapat, yeni proxy al, yeniden baglan
Hata Yonetimi
Baileys 'connection.update' event'ini dinle: ban, logout, network error ayirt et
Ban durumunda: hesabi banned olarak isaretele, otomatik yedegi devreye al
Network hatalarinda: 3 kez retry, her seferinde 30sn bekle
BullMQ job basarisiz olursa: 3 kez tekrar dene, sonra failed olarak kaydet
